# Six10 Pricing Strategy Engine

Automated weekly pricing recommendation engine for Six10 Ventures' Amazon brands. Every Monday it pulls live data from Google Drive, computes 9 signals per ASIN, generates RAISE / LOWER / HOLD recommendations with LLM reasoning, writes a 4-tab Google Sheet, and sends a stakeholder HTML email.

**Brands covered:** AquaDoc, Mokita Naturals, NL Brands (PawMedica / Ruckus / Pure Velvet), Pureauty Naturals, VisiVite (~185 active ASINs)

---

## How it works

```
Google Drive
  └─ Sellerise CSVs, FBA inventory, COGS sheet, Helium10, fee previews, returns
        │
        ▼  drive_sync.py
  Data Feeds/   (local cache, not in repo)
        │
        ▼  data_loader.py
  signal_computer.py  →  9 signals per ASIN (CTR, conversion, velocity, margin,
                          trend, refund rate, inventory pressure, YoY, BSR)
        │
        ▼  recommendation.py
  RAISE / LOWER / HOLD / LOWER_BLOCKED / LOWER_VELOCITY_DEFENCE
        │
        ▼  margin_calculator.py + llm_layer.py
  Dollar impact scenarios  +  Claude Sonnet reasoning
        │
        ├──▶  sheets_writer.py   →  Google Sheets (Summary / Full Catalog / History / Pipeline Actions)
        └──▶  email_reporter.py  →  Weekly HTML email (Gmail API)
```

## GitHub Actions — automated weekly run

The workflow runs every **Monday at 8 am EST** (cron `0 13 * * 1`). It can also be triggered manually from the Actions tab.

### Required secrets

Go to **Settings → Secrets and variables → Actions → New repository secret** and add these four:

| Secret name | What to paste |
|---|---|
| `ANTHROPIC_API_KEY` | Your Anthropic API key (`sk-ant-api03-…`) |
| `GOOGLE_SHEETS_CREDENTIALS_JSON` | Full contents of `pricing-strategy-engine-4983eab4d919.json` |
| `GMAIL_CREDENTIALS_JSON` | Full contents of `credentials-gmail-pricing.json` |
| `GMAIL_TOKEN_JSON` | Full contents of `gmail_token.json` |

> The credential JSON files live on your machine and are excluded from the repo by `.gitignore`. Paste the complete file contents (including all braces and quotes) as the secret value.
>
> The Gmail refresh token inside `gmail_token.json` does not expire under normal use. The engine auto-renews the short-lived access token on every run — no secret rotation needed.

## Local setup

### 1. Install dependencies

```bash
python -m venv .venv
# Windows:  .venv\Scripts\activate
# Mac/Linux: source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Credentials

Place the following files in the repo root (they are gitignored):

- `pricing-strategy-engine-4983eab4d919.json` — Google service account (Sheets + Drive)
- `credentials-gmail-pricing.json` — Gmail OAuth client credentials
- `gmail_token.json` — Gmail OAuth token (generated by `authorize_gmail.py`)
- `.env` — Anthropic API key:
  ```
  ANTHROPIC_API_KEY=sk-ant-api03-…
  ```

### 3. First-time Gmail auth

If `gmail_token.json` doesn't exist yet:

```bash
python authorize_gmail.py
```

Follow the browser prompt. The token file is created automatically.

### 4. Run

```bash
# Full run (Drive sync → signals → Sheets write → email)
python main.py

# Compute only, no writes
python main.py --dry-run

# Sheets write, no email
python main.py --no-email

# Email only (re-send last output)
python main.py --email-only

# Write to local Excel instead of Sheets
python main.py --dry-run --output-local test_output.xlsx
```

## Configuration

All tunable parameters are in `config.yaml`:

- **Brand margin floors** and **price elasticity** per brand
- **Signal weights** (CTR, conversion, velocity, margin, trend, refund rate, inventory pressure, YoY, BSR)
- **Google Drive folder IDs** for each data feed
- **Google Sheet ID** for output
- **Email recipients**
- **LLM model** and token budget

## Apps Script setup

`apps_script.js` powers the **Approve / Skip / Dismiss** buttons in the weekly email.

1. Open [script.google.com](https://script.google.com) → New project
2. Paste the contents of `apps_script.js`
3. Deploy → New deployment → Web app → Execute as: Me → Anyone
4. Copy the deployment URL and paste it into `config.yaml → email.dismiss_script_url`

To add the **Approve All** batch handler, redeploy as a **new version** (not a new deployment) so the URL stays the same.

## Signals reference

| Signal | Weight | Source |
|---|---|---|
| CTR vs brand average | 10% | Sellerise (ad-active days only) |
| Conversion vs 90-day baseline | 20% | Sellerise |
| Velocity vs prior 30 days | 20% | Sellerise |
| Structural margin vs floor | 15% | Computed (price − COGS − FBA − referral) |
| Net-profit trend (regression) | 15% | Sellerise |
| Refund rate | 5% | Amazon Return Reports |
| Inventory pressure (days of supply) | 5% | FBA Inventory snapshot |
| YoY velocity factor (seasonal brands) | 5% | Sellerise historical |
| BSR / keyword rank trend | 5% | Helium10 |
